# 🎉 终于等到你！

今天是个值得纪念的日子。我的个人独立博客，终于在 `songjiayi11.github.io` 这个地址安家了。

这看起来只是一个普通的网页，但为了让它能“活”过来，我刚刚经历了一段相当“酸爽”的折腾之旅。

## 🛠️ 那些年我踩过的坑

作为纯纯的新手，刚开始的我天真地以为，建个博客也就是点几下鼠标、填几个字的事。结果现实狠狠地给我上了一课：

1.  **报错 404 的绝望：** 好不容易把文件都传上去了，满心欢喜地刷新网页，结果看到的是一片空白和巨大的 "404 Not Found"。那种感觉，就像满怀期待地打开礼物盒，结果里面什么都没有。
2.  **白屏与 Theme 的诅咒：** 404 好容易解决了，网页又变成了“白板”。我开始试图通过修改 `_config.yml` 来加个漂亮的主题。然而，仅仅因为写错了一个配置，GitHub 的系统就立刻罢工，红色的报错叉号（❌）在 `Actions` 里一次比一次扎眼。
3.  **卡在缓存里的鬼打墙：** 最让我崩溃的是，有时候明明觉得改对了，但页面怎么刷新都是旧的错误状态。最后才知道，是浏览器缓存和 GitHub 的缓存“联手”给我使绊子。按 `Ctrl + F5` 强制刷新，甚至要开“无痕模式”才能看到真相。

## 🧠 经历之后的感悟

虽然过程很不顺，甚至几次气得想关掉电脑，但现在看着屏幕上这只言片语，我觉得一切都值了。

或许我现在的网站排版还很简陋，没有漂亮的导航栏，没有酷炫的特效。但它代表着我的 **"0 到 1"**。它告诉我，那些晦涩的报错日志不是用来吓人的，而是一个个通往成功的线索。

## 🚀 未来的计划

既然这块“硬骨头”已经啃下来了，接下来我想慢慢完善它：

*   **学会换皮肤：** 下次要小心翼翼地尝试外部主题，不会再像今天一样盲目地改代码了。
*   **多写文章：** 我会把这个博客当做我的笔记本，记录下学到的代码知识、生活里的点滴，以及更多像今天这样“折腾”出来的故事。

最后，感谢那个没有放弃的自己。如果你也正在为建站报错而苦恼，请相信我，坚持下去，你一定能看到属于你的那行字！

**—— 记录于 2026年7月20日，我的博客正式上线第一天。**
